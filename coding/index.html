Mkdocs

This site has been built using mkdocs.org., sofor full documentation visit mkdocs.org. Some useful tips and reference material can be found at Material for MkDocs, such as how to add tables, images, formatting, and so on.

A starting guide can be found here.

Useful commands

I usually build my local site in my hard drive, and later commit changes using GitHub.

  • mkdocs new [dir-name] - Create a new project.
  • mkdocs serve - Start the live-reloading docs server.
  • mkdocs build - Build the documentation site.
  • mkdocs -h - Print help message and exit.

Project layout

By creating a new project, the configuration will be as follows:

mkdocs.yml    # The configuration file.
docs/
    index.md  # The documentation homepage.
    ...       # Other markdown pages, images and other files.
